                        # 日常笔记-编React技术栈

                        # let vs const vs var

                        var:

                        var声明的变量是函数作用域，会变量提升。

                        let、const:

                        • let和const声明的变量是块级作用域；
                        • 不存在变量提升；
                        • 一个作用域中不允许重复声明；
                        • 先声明后使用；
                        • const声明的变量所指向的内存地址不得改动。

                        # for...of与for...in的区别

                        for...in语句以任意顺序迭代对象的可枚举属性(包括原型),==取键==；

                        for...of语句遍历可迭代对象定义的迭代数据。常见于数组、字符串等，==取值==。

                        以下示例显示了与Array一起使用时，for...of循环和for...in循环之间的区别。

                        Object.prototype.objCustom = function() {}; 
Array.prototype.arrCustom = function() {};

let iterable = [3, 5, 7];
iterable.foo = 'hello';

for (let i in iterable) {
  console.log(i); // 0, 1, 2, "foo", "arrCustom", "objCustom"
}

for (let i in iterable) {
  if (iterable.hasOwnProperty(i)) {
    console.log(i); // 0, 1, 2, "foo"
  }
}

for (let i of iterable) {
  console.log(i); // 3, 5, 7
}

                        # React生命周期函数的理解

                        挂载阶段

                        • constructor()
                        • static getDerivedStateFromProps()
                        • render()
                        • componentDicMount()

                        更新阶段

                        • static getDerivedStateFromProps()
                        • shouldComponentUpdate()
                        • render()
                        • getSnapshotBeforeUpdate()
                        • componentDidUpdate()

                        卸载阶段

                        • componentWillUnmount()

                        错误处理

                        • static getDerivedStateFromError()
                        • componentDidCatch()

                        # constructor()

                        • 如果不初始化state或不进行方法绑定时，可以不为React组件实现构造函数。
                        • 构造函数中不要调用setState()方法。
                        • 不可以在构造函数中引入任何副作用或订阅。如果遇到，请将对于的操作放置到componentDidMount中。

                        警告

                        避免将props的值赋值给state！以下是个常见的错误：

                        constructor(props) {
 super(props);
 // 不要这样做
 this.state = { color: props.color };
}

                        只有在你刻意忽略 prop 更新的情况下使用。

                        # render()

                        render()是class组件中唯一必须实现的方法。

                        render()的返回值：

                        • React元素。无论是<div/>还是<MyComponnet/>都是React元素。
                        • 数组或fragments。数组项可以是jsx。
                        • Portals。可以渲染子节点到不同的 DOM 子树中。
                        • **字符串或数字类型。**它们在 DOM 中会被渲染为文本节点。
                        • 布尔值或null。什么都不渲染。（主要用于支持返回 test && <Child/> 的模式，其中 test 为布尔类型。)

                        render()函数应该为纯函数。

                        警告

                        如果 shouldComponentUpdate() 返回 false，则不会调用 render()。

                        # componentDidMount()

                        componentDidMount() 会在组件挂载后（插入 DOM 树中）立即调用。依赖于 DOM 节点的初始化应该放在这里。如需通过网络请求获取数据，此处是实例化请求的好地方。

                        这个方法是比较适合添加订阅的地方。如果添加了订阅，请不要忘记在 componentWillUnmount() 里取消订阅

                        # componentDidUpdate()

                        componentDidUpdate(prevProps, prevState, snapshot)
// snapshot 为getSnapshotBeforeUpdate()的返回值

                        当组件更新后，可以在此处对 DOM 进行操作。如果你对更新前后的 props 进行了比较，也可以选择在此处进行网络请求。（例如，当 props 未发生变化时，则不会执行网络请求）。

                        componentDidUpdate(prevProps) {
  // 典型用法（不要忘记比较 props）：
  if (this.props.userID !== prevProps.userID) {
    this.fetchData(this.props.userID);
  }
}

                        你也可以在 componentDidUpdate() 中直接调用 setState()，但请注意它必须被包裹在一个条件语句里，正如上述的例子那样进行处理，否则会导致死循环。

                        # componentWillUnmount()

                        componentWillUnmount() 会在组件卸载及销毁之前直接调用。在此方法中执行必要的清理操作，例如，清除 timer，取消网络请求或清除在 componentDidMount() 中创建的订阅等。

                        componentWillUnmount() 中不应调用 setState()，因为该组件将永远不会重新渲染。组件实例卸载后，将永远不会再挂载它。

                        # 错误处理

                        错误边界在开发模式下只会闪一下，界面渲染依然会中断。在生产模式下不会中断界面渲染。

                        # React 中 keys 的作用是什么？

                        文档

                        • key 帮助 React 识别哪些元素改变了，比如被添加或删除。减少组件的更新区域，提升性能
                        • 一个好的经验法则是：在 map() 方法中的元素需要设置 key 属性。
                        • key是唯一的，是指key 只是在兄弟节点之间必须唯一。
                        • key 会传递信息给 React ，但不会传递给你的组件。即，props.key读不到值。

                        # 调用 setState 之后发生了什么？

                        • setState() 将对组件 state 的更改加入更新队列，并通知 React 需要使用更新后的 state 重新渲染此组件及其子组件。
                        • setState() 并不是立即更新组件。它会批量推迟更新。

                        # 简述 flux 思想

                        答案：Flux 的最大特点，就是数据的"单向流动"。

                        1. 用户访问 View
                        2. View 发出用户的 Action
                        3. Dispatcher 收到 Action，要求 Store 进行相应的更新
                        4. Store 更新后，发出一个"change"事件
                        5. View 收到"change"事件后，更新页面

                        # React中组件传值方式

                        • props和回调。常用用于父子组件通信。
                        • context对象。常用于跨级组件通信。
                        • redux、mobx等状态库。
                        • 事件发布订阅(node的events内置库)。不常用，但是是个很重要的思想。

                        # 开启React项目装饰器模式

                        1. 依赖react-app-rewired、customize-cra两个库。

                        2. 修改package.json文件

                          "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test",
    "eject": "react-scripts eject"
  },
      
// 改成如下：
  "scripts": {
    "start": "react-app-rewired start",
    "build": "react-app-rewired build",
    "test": "react-app-rewired test",
    "eject": "react-scripts eject"
  },

                        3. 在项目根目录新建config-overrides.js,用于修改默认配置

                        4. 在config-overrides.js中添加如下配置以启动装饰器模式

                          const {
    override,
    addDecoratorsLegacy
} = require("customize-cra");

module.exports = override(
    addDecoratorsLegacy(),
);

// 这仅仅只是添加装饰器，在这儿还可以添加很多其他功能，根据需要添加。

<React.Suspense fallback={<div>Loading...</div>}>
    <LazyComponent />
</React.Suspense>
